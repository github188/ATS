关于测试驱动和测试点的说明


1. 测试驱动表示具体的一款设备的测试平台，表示此台设备的测试全过程。

2. 测试点表示设备测试过程中的一个测试过程，通常，一个测试驱动由多个测试点组成。


例子：

系统支持的测试点：(7个)
    TP_1, TP_2, TP_3, TP_4, TP_5, TP_6, TP_7
    
    
DEV_A设备的测试驱动为 TDRV_A：假设A需要测试TP_1, TP_5, TP_7三个点，则
    TDRV_A 为TP_1, TP_5, TP_7的集合。
    

DEV_B设备的测试驱动为TDRV_B：假设B需要测试TP_1, TP_2, TP_3, TP_5四个点，则
    TDRV_B 为TP_1, TP_2, TP_3, TP_5的集合。
    

不同设备利用系统支持的测试点组成此设备的测试驱动，然后进行测试。
