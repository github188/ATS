关于设备探测插件


1. 对于每一类设备来说，通常都有类似的检测设备方法，例如，IPC设备通常可以通过Onvif协议来自动发现设备，对于IPC设备来说，探测器可以通过发送Onvif的Hello方法来检测系统中的IPC设备，前提时IPC必须支持Onvif。

2. 当探测器检测到设备时，构造出一个ATS_Device设备，然后把此设备注册道系统中，系统就知道了此设备的存在，就可以进行后续的操作。

3. 某些时候，系统的的设备可能因为某些原因断开，比如人为关闭设备等，这是就要保证让系统能够知道此设备已经断开，从而防止后续非法操作，所以，必须要能够动态的知道设备的状态，可以通过心跳方法来检测设备，即是通过在一定时间内再次检测设备来活得设备的状态，当发现新设备时，把新设备注册道系统中，当发现原有的设备断开时，把此设备从系统中卸载。

4. 探测器模块作为一个插件，有利于后续升级及添加新的探测器，从而集成新的设备检测。
